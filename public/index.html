What is Divshot

Divshot define themselves as "Static Web Hosting for Developers" with the tagline "Blazing fast CDN, simple CLI deployment, hassle-free test environments. And much more.". Personally I think of it as Heroku for static web apps.

For me, what makes life easier with Divshot, is that it's easy to setup (GitHub auth and free tier), easy to integrate with CI (Travis support and environments), and extra services (ajax proxy, email, prerenders). Of course it's not for everything, but if you just want to get some simple app or prototype running somewhere, without having to worry too much about deployment and hosting, then it's kind of nice.

Setup

Tools and environment


          npm install -g divshot-cli
          divshot login

Repo setup

Adding divshot.json


          divshot init

Travis CI setup


          touch .travis.yml
          divshot auth:token
          travis setup divshot

If you have some build steps you'll want to add skip_cleanup: true to the deployment section of the config.

If you want per-branch environments, you can add the following to .travis.yml 


          deploy:
            environment:
              master: development
              staging: staging
              production: production

Deployment

Local deployment


          divshot server
          open http://localhost:3474

This runs a local development server with configuration taken from divshot.json. So root paths and URL cleanups and such work in the same way as in production.

Note that in order to use the API services, such as AJAX proxy and email, you need to isntall dependencies and specify them separately in commandline arguments.

Deploying to Divshot


          divshot deploy development
          open http://development.divshot-demo.divshot.io/

This only pushes files under the root directory defined in divshot.json, with additional excludes possible. So no need to worry about pushing large amounts of dependencies or temporary files to the server.

Cool features

Custom routing

You can point multiple paths to a single file, by specifying routes in divshot.json. This page only has a single file, so we can just route every url to the same file with: 


          {
            "routes": {
              "/**": "index.html"
            }
          }

So now a link to /no-way-this-page-exsists serves this same page.

Of course we could just have user error.html for this trick. For more info on custom routings check out https://docs.divshot.com/guides/routing.

AJAX Proxy

Divshot provides a nice AJAX proxy for connection to public APIs that for some reason don't provide CORS headers. Configure your proxy address in divshot.json and connect to the proxy at the path /__/proxy/{$NAME/{$PATH}.

Note, if you want to run the proxy on your local dev server, you need to do 


          npm install superstatic-proxy
          divshot server --services proxy

Example: Fetching data for road temperatures

Foreca provides some realtime data on road weather conditions through a public API at http://tie.digitraffic.fi/sujuvuus/ws/roadConditions, unfortunately we can't connect to it directly from the browser, due to same-origin policy.

Adding the following proxy config to divshot.json, and we can access the API at /__/proxy/digitraffic/sujuvuus/ws/roadConditions 


          {
            "proxy": {
              "digitraffic": {
                "origin":"http://tie.digitraffic.fi/",
                "cookies": false
              }
            }
          }

Note: To use this on local development you need to do

Environment variables

Divshot also supports environment variables which are injected through a JavaScript file served at /__/env.js. Alternatively, they are available as JSON at /__/env.json. These environment variables are user defined, and also contain build information. Could be useful for configuring API endpoints and such.

Techincal information

Hosting

Currently Divshot apps are hosted on AWS Elastic Beanstalk instances, with MaxCDN providing a CDN for all static files.

Server

Divshot pages are served by superstatic, which is a custom server for serving Divshot apps. Implemented in node.js using Express, providing basic features such as routing. Additional features are added as modular services, such as ajax proxy and email sending.
